## 二、技术路线

```
UAV 图像 → ① 图像检索（CAMP, ConvNeXt-Base, 384×384, 10ms/张）
         → ② 像素匹配（RoMa, DINOv2-ViT-L, 659ms/张）
         → ③ PnP 求解（P3P + RANSAC, OpenCV, 1ms/张）
         → 输出: 无人机经纬度
```

最佳组合：CAMP（检索）+ RoMa（匹配）+ Top N Re-rank（定位策略）

- 问题：RTX 3080 (10GB) 跑 RoMa 时 OOM
- 解决：降低 RoMa 分辨率 coarse_res 560→280, upsample_res 864→512
- 效果：模型显存从 ~4GB 降到 ~0.4GB
- 代码修改：`code/Matching_Models/RoMa/demo/Roma_match.py` 第 16-17 行

## 七、关键技术点

### 7.1 CAMP 检索模型

- Backbone: ConvNeXt-Base
- 预训练: University-1652 数据集
- 输入: 384×384
- 核心思想: 同一 training batch 内比较同一平台不同场景，增加负样本
- R@1: 82.7%

### 7.2 RoMa 匹配模型

- Backbone: DINOv2-ViT-L
- 类型: 密集匹配
- PnP 求解器: P3P + RANSAC（OpenCV）
- A@5m: 70.1%（论文）
- 速度: 659ms/帧

### 7.3 代码修改记录

1. `code/Retrieval_Models/CAMP/get_CAMP.py` 第133行：
   - 原：`args = parser.parse_args(namespace=self)`
   - 改：`args = parser.parse_args([], namespace=self)`
   - 原因：避免与 Baseline.py 的 argparse 冲突

2. `code/Matching_Models/RoMa/demo/Roma_match.py` 第16-17行：
   - 原：`roma_outdoor(device=device, weights=model_path, dinov2_weights=dinov2_path)`
   - 改：`roma_outdoor(device=device, weights=model_path, dinov2_weights=dinov2_path, coarse_res=280, upsample_res=512)`
   - 原因：降低显存占用，适配 RTX 3080

3. `code/utils.py`：
   - 添加 `MATCH_REGISTRY` 字典，支持多种匹配方法
   - 添加 `SIFT_Init` 和 `SIFT_match` 导入
   - 在匹配循环中添加 `torch.cuda.empty_cache()`

4. `code/Matching_Models/SIFT/SIFT_match.py`：
   - 新增 SIFT 匹配方法实现
