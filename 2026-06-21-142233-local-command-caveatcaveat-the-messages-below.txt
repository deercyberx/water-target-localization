● 两种方案对比：

  ┌──────────┬─────────────────────────┬────────────────────┐
  │   维度   │       python-docx       │       poi-tl       │
  ├──────────┼─────────────────────────┼────────────────────┤
  │ 语言     │ Python（现有环境）      │ Java（需额外环境） │
  ├──────────┼─────────────────────────┼────────────────────┤
  │ 模式     │ 代码生成文档            │ 模板+数据渲染      │
  ├──────────┼─────────────────────────┼────────────────────┤
  │ 项目经验 │ ✅ 已用过（报告v2脚本） │ ❌ 无              │
  ├──────────┼─────────────────────────┼────────────────────┤
  │ 图表插入 │ 直接读取已生成的png     │ 需要配置模板变量   │
  ├──────────┼─────────────────────────┼────────────────────┤
  │ 表格生成 │ 代码控制行列            │ 模板循环           │
  ├──────────┼─────────────────────────┼────────────────────┤
  │ 灵活度   │ 高（每段可单独控制）    │ 中（依赖模板结构） │
  ├──────────┼─────────────────────────┼────────────────────┤
  │ 维护性   │ 代码即文档              │ 模板+代码分离      │
  └──────────┴─────────────────────────┴────────────────────┘

  1. 环境现成，不需要装Java

  3. 图表和表格数据都在Python生态内，直接读取
  4. 提纲的行列结构用代码控制更精确

  用 poi-tl 的唯一优势是有现成的 .docx
